<objective>
Capture body composition baseline at member admission. Add an "Initial Measurements" section
to the Add Member form with six fields: Weight (kg), Height (cm), BMI (auto-calculated,
read-only), Body Fat %, Waist (cm), and Chest (cm).

When a brand-new member is saved and any measurement is filled in, the submit handler
automatically creates a `progress_records` document dated on the member's join date. This
seeds the progress chart from Day 1 without requiring the gym owner to visit the Progress
module separately.

Editing an existing member pre-fills the measurement fields from the saved document and
updates them on save, but does NOT create a second progress record (idempotent on edit).
</objective>

### Task 2 — BMI auto-calculation

**File:** `modules/members.js`
**Wave:** 1 (can run alongside Task 1)

Add a helper function `calcBmi(weightKg, heightCm)` near the top of the module (before the
export):

```js
function calcBmi(weightKg, heightCm) {
  const w = parseFloat(weightKg);
  const h = parseFloat(heightCm) / 100;
  if (!w || !h || h <= 0) return "";
  return (w / (h * h)).toFixed(1);
}
```

In `bind()`, after the existing `form.startDate.addEventListener` block, attach input listeners:

```js
function updateBmi() {
  form.initBmi.value = calcBmi(form.initWeight.value, form.initHeight.value);
}
form.initWeight?.addEventListener("input", updateBmi);
form.initHeight?.addEventListener("input", updateBmi);
```

**Acceptance:** Typing weight=80, height=175 auto-fills BMI=26.1. Clearing either field clears BMI.

### Task 4 — Persist measurements on the member document + auto-create initial progress record

**File:** `modules/members.js`
**Wave:** 2 (depends on Task 1 fields existing)

Modify the `form.addEventListener("submit", ...)` handler. After `const payload = formData(form);`
and before the `withButtonLoading` call:

1. **Extract measurement fields** from the payload into a separate object, but leave them in
   `payload` so they are stored on the `members` document:

```js
const isNew = !payload.id;
const measurements = {
  weight: payload.initWeight || "",
  height: payload.initHeight || "",
  bmi:    payload.initBmi    || "",
  bodyFat: payload.initBodyFat || "",
  waist:  payload.initWaist  || "",
  chest:  payload.initChest  || ""
};
const hasMeasurements = Object.values(measurements).some(v => v !== "");
```

2. **Inside `withButtonLoading`**, after `context.services.data.save(collections.members, payload)`
   returns `saved`, conditionally create the progress record:

```js
const saved = await context.services.data.save(collections.members, payload);
if (isNew && hasMeasurements) {
  await context.services.data.save(collections.progress, {
    memberId: saved.id,
    date: payload.joinDate || today(),
    weight:  measurements.weight,
    bmi:     measurements.bmi,
    bodyFat: measurements.bodyFat,
    waist:   measurements.waist,
    chest:   measurements.chest,
    notes:   "Initial admission measurement"
  });
  context.applyChange(collections.progress, {
    memberId: saved.id,
    date: payload.joinDate || today(),
    ...measurements,
    notes: "Initial admission measurement"
  });
}
```

Note: `height` is NOT a field in `progress_records` (the progress schema tracks body composition
metrics, not anthropometric constants). Store height only on the member document.

**Acceptance:**
- Add a new member with weight=75, height=170 → a progress_records document is created with
  memberId, date=joinDate, weight=75, bmi=26.0, and notes="Initial admission measurement".
- Edit that member and re-save → no duplicate progress record is created.
- Add a new member with no measurements → no progress record is created.
