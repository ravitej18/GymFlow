route ("My Members", trainer-only). The module lets a trainer see the members assigned to them,
assign a workout template to a member (writes `workout_assignments`), and write today's session for
a member (writes `workout_sessions`).

daily session with exercises/notes). This is the producer side of the trainer→member workflow — the
member consumption UI (07-03) reads the records this plan creates.

modules map, nav entry); `scripts/smoke-test.mjs` covers the new module for trainer and
trainer-pending contexts.

|----------|-------------|
| trainer client → Firestore | Trainer browser writes workout_sessions / workout_assignments; payload fields are user-controllable |
| rendered HTML → DOM | Member/template names and free-text exercises/notes are interpolated into innerHTML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Tampering | trainerId in save payload | mitigate | UI sets `trainerId: me.id` from the trainer's own roster doc; Firestore rule (07-01) re-validates uid match server-side |
| T-07-06 | Information Disclosure | members of other trainers | mitigate | UI lists only `members` where `assignedTrainer === me.id`; member/template selects are sourced from that filtered list |
| T-07-07 | Tampering (XSS) | exercises/notes/name rendering | mitigate | All interpolated fields wrapped in `escapeHtml()` before innerHTML, matching existing modules |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No package installs — pure ES module, zero new dependencies |
</threat_model>
