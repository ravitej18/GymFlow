Also update `memberStatus()` to recognise the stored "Paused" status. The pause/resume
UI (Plan 09-02) will write `member.status = "Paused"` on pause and clear it on resume.
Insert the Paused check immediately after the Pending check:

```js
export function memberStatus(member) {
  if (member.status === "Pending")   return "Pending";
  if (member.status === "Paused")    return "Paused";
  if (member.status === "Suspended") return "Suspended";
  const remaining = daysUntil(member.endDate);
  if (remaining < 0)   return "Expired";
  if (remaining <= 15) return "Expiring Soon";
  return "Active";
}
```

**Acceptance:** `collections.membershipPauses === "membership_pauses"`.
`memberStatus({ status: "Paused" })` returns `"Paused"`.

### Task 4 — Firestore rules for membership_pauses

**File:** `firestore.rules`

Add a `match /membership_pauses/{pauseId}` block. Rules:
- Owner in the same gym can create, update, delete pause records
- Any signed-in same-gym user (member) can read (so member portal can show pause history if needed)

Add directly after the `match /progress_records/` block (or after `match /reminders/`
— wherever the last collection block appears before the closing braces):

```
match /membership_pauses/{pauseId} {
  allow read: if sameGym();
  allow create, update: if owner() && incomingSameGym();
  allow delete: if owner() && sameGym();
}
```

**Acceptance:** Rules file contains `match /membership_pauses/` with read and write
blocks using the existing `sameGym()`, `owner()`, and `incomingSameGym()` functions.

In `bind()`, add a submit handler for `#pause-limits-form`:

```js
const pauseLimitsForm = root.querySelector("#pause-limits-form");
pauseLimitsForm?.addEventListener("submit", async (event) => {
  event.preventDefault();
  const payload = formData(pauseLimitsForm);
  payload.maxPausesPerYear = Number(payload.maxPausesPerYear);
  payload.maxPauseDays     = Number(payload.maxPauseDays);
  await context.services.data.saveSettings(payload);
  context.toast("Pause limits saved.");
  await context.refreshView();
});
```

### Task 7 — Renewal queue: exclude Paused members

**File:** `modules/renewals.js`

The `watched` list currently includes members expiring within 30 days or expired.
Paused members have their endDate extended, so they may appear incorrectly in the
queue. Add a filter to exclude them:

Current:
```js
const watched = members
  .map(...)
  .filter((member) => member.remaining <= 30 || member.computedStatus === "Expired")
  .sort(...);
```

Change the filter to:
```js
.filter((member) =>
  member.computedStatus !== "Paused" &&
  (member.remaining <= 30 || member.computedStatus === "Expired")
)
```

**Acceptance:** A member with `status: "Paused"` does not appear in the Renewals queue.
